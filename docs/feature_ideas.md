# Feature Ideas & Brainstorming

> 이 문서는 프로젝트의 미래 발전 방향과 추가 기능 아이디어를 정리하는 공간입니다.
> 사용자의 제안을 바탕으로 구체화된 기획을 기록합니다.

## 1. 콘텐츠 추천 및 탐색 (Content Discovery)

### 관련 표현 추천 (Related Expressions)

- **위치**: 표현 상세 페이지(`app/expressions/[id]/page.tsx`) 하단.
- **방식**:
  - **카테고리 기반**: 현재 표현과 동일한 `category`를 가진 다른 표현들을 랜덤으로 3~4개 노출.
  - **태그 기반**: 현재 표현의 태그 중 하나 이상을 공유하는 표현 추천.
- **목적**: 사용자가 관심 있는 주제의 다른 표현을 자연스럽게 탐색하도록 유도 (체류 시간 증대).

### 리스트 탐색 경험 (List Navigation)

- **문제**: 데이터가 수백 개 이상 쌓일 경우, 한 페이지에 모든 카드를 로드하면 성능 저하 및 스크롤 피로감 발생.
- **해결 방안**: **'더 보기(Load More)' 버튼** 방식 도입.
  - 초기에는 최신순 12~24개 로드.
  - 하단 '더 보기' 버튼 클릭 시 다음 12개 추가 로드 (Client-side Fetching).
  - **장점**: 무한 스크롤의 '푸터 접근 불가' 문제 해결 및 페이지네이션의 '클릭 불편함' 사이의 균형 유지.

### 아카이브 / 인덱스 페이지 (Archive & Index)

- **개요**: '더 보기' 방식의 탐색 불편함(원하는 표현을 찾기 어려움)을 해소하기 위한 **표현 전용 색인 페이지**.
- **구성**:
  - **A-Z 인덱스**: 알파벳 순으로 표현(Expression)만 텍스트 리스트로 나열.
  - **최신/역순 인덱스 (Upload Order)**: 업로드된 시간 순서대로 나열하여, 전체 발행 흐름을 한눈에 파악.
  - **월별 아카이브**: 2026년 1월, 2월 등 생성 시기별로 모아보기.
  - **컴팩트 뷰**: 카드 형태가 아닌, 엑셀 표나 심플한 리스트(`Expression` - `Meaning`) 형태로 한눈에 많은 정보를 스캔할 수 있도록 구성.
- **목적**: 사용자가 "아, 그때 그 표현 뭐였지?" 하고 빠르게 찾고 싶을 때 유용한 '사전(Dictionary)' 같은 역할 수행.

## 2. 퀴즈 모드 (Quiz Mode)

### '오늘의 미션' 전용 페이지

- **개요**: 단순히 표현을 읽는 것을 넘어, 능동적으로 학습할 수 있는 퀴즈 전용 인터페이스.
- **기능**:
  - 하루에 한 번, 혹은 랜덤으로 여러 문제 풀기.
  - 객관식(의미 맞추기), 빈칸 채우기(문장 완성), 받아쓰기 등 다양한 유형.
- **UX**: 게임화(Gamification) 요소를 도입하여 성취감 부여 (연속 정답 콤보 등).

## 3. 인터랙션 및 저장 (Interaction & Save)

### 저장 및 스크랩 (Save & Scrap)

- **개념 구분**:

  - **하트 (Like/Save Item)**:

    - **대상**: 개별 영어 표현 (Single Expression).
    - **동작**: 클릭 시 '내 좋아요 목록' 혹은 기본 '내 단어장'에 해당 표현이 저장됨.
    - **목적**: 나중에 다시 보고 싶은 표현을 가볍게 수집.

  - **북마크 (Bookmark/Fork List)**:

    - **대상**: 단어장 (Vocabulary List / Collection).
    - **동작**: 다른 사용자나 공식 계정이 만든 단어장 전체를 내 라이브러리로 가져오거나 구독함.
    - **목적**: 유용한 큐레이션(예: "여행 필수 표현 50선")을 통째로 학습하기 위함.

## 4. 커스텀 단어장 (Custom Vocabulary)

### 사용자 주도 학습 (User-Driven Learning)

- **개인 단어장**: 사용자가 원하는 단어/표현을 선택하여 자신만의 단어장을 생성.
- **학습 모드**:
  - **View**: 리스트 형태로 보기.
  - **Learn**: 뜻 가리기, 예문 보기 등 학습 모드.
  - **Flashcard**: 앞면(영어)/뒷면(뜻) 뒤집기 방식의 암기 카드.
- **사용자 추가**: 서비스에서 제공하는 표현 외에, 사용자가 직접 외부에서 배운 단어를 추가하는 기능.

## 5. 커뮤니티 및 소셜 (Community & Social)

### 단어장 공유 및 구독

- **공개 설정**: 단어장 생성 시 `Public` / `Private` 설정 가능.
  - **Public**: 다른 사용자들도 검색하고 볼 수 있음.
  - **Private**: 나만 볼 수 있음.
- **구독(Follow)**: 유용한 단어장을 만든 사용자를 팔로우하거나, 특정 단어장을 '구독'하여 업데이트 받아보기.
- **소셜 프로필**:
  - 내가 만든 공개 단어장 목록.
  - 다른 사용자가 내 단어장을 저장(Fork/Like)한 횟수 표시 (명예욕 자극).
- **레퍼런스**: 유튜브 뮤직 재생목록, Spotify 플레이리스트 공유 방식 참고.
