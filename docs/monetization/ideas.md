# Monetization Ideas & Concepts

> 이 문서는 수익화 관련 브레인스토밍 및 전략적 조언(`docs/monetization/brainstorming.md`)을 정리한 문서입니다.
> 실제 구현 로드맵은 `docs/monetization/strategy.md`를 참고하세요.

## 1. 핵심 철학: "From Blog to SaaS"

단순히 정보를 제공하는 블로그가 아니라, **"사용자의 학습 데이터를 축적하고 최적화해주는 시스템"**으로 진화해야 수익화가 가능합니다.

- **기능이 아닌 '전환 트리거'에 집중**: 사용자는 기능 자체가 아니라, "불편함이 해소되는 순간"에 지갑을 엽니다.
  1. 단어를 자꾸 까먹을 때 (망각)
  2. 단어장이 감당 안 될 때 (관리)
  3. 실력 향상이 안 느껴질 때 (성장)
  4. 나만의 학습 루트가 필요할 때 (개인화)

## 2. 수익 모델 (Business Model)

### 모델: Freemium + Monthly Subscription

- **가격**: 월 $9.99 (심리적 저항이 낮은 구간)
- **결제 수단**: PayPal (글로벌 표준)

### 무료 vs 유료 차별화 전략 (Hybrid Storage)

| 구분            | 무료 사용자 (Free)              | 유료 사용자 (Pro)                  |
| :-------------- | :------------------------------ | :--------------------------------- |
| **저장소**      | **Local Storage** (브라우저)    | **Supabase DB** (클라우드)         |
| **데이터 보존** | 기기 변경/캐시 삭제 시 **소멸** | 영구 보존 & **멀티 디바이스 연동** |
| **AI 기능**     | 제한적 체험                     | 무제한 (개인화 예문, 오답 분석)    |
| **핵심 가치**   | 찍먹 (Trial)                    | **자산화 (Asset)** & 안심          |

> **전환의 핵심 순간 (Migration Moment)**:
> 사용자가 "이 데이터 날리면 안 되는데..."라고 느끼는 순간, "로그인하고 안전하게 보관하세요"라고 제안하여 자연스럽게 DB로 데이터를 이관(Sync)시킵니다.

## 3. 기능 고도화 단계 (Evolution Steps)

### Level 1: 데이터 수집 (MVP)

- **기능**: 회원가입, 단어 저장, 단어장 생성.
- **목표**: 유저가 "영어 흔적"을 남기게 하기.

### Level 2: 플래시카드 자동화

- **기능**: 저장한 단어로 카드 자동 생성 (뜻, 기본 예문).
- **목표**: "공부가 된다"는 효용감 제공.

### Level 3: 제한 및 불편 설계 (Soft Lock)

- **기능**: 저장 개수 제한, 커스텀 제한.
- **메시지**: "여기까지는 무료입니다. 더 체계적으로 관리하려면 업그레이드하세요."

### Level 4: 기억 기반 학습 (Retention)

- **기능**: 망각 곡선 기반 복습(SRS), 틀린 단어 집중 공략.
- **가치**: "나를 관리해주는 서비스"라는 느낌.

### Level 5: AI 개인화 (Profit)

- **기능**: 내 단어로 문장 만들기, 개인별 오답 분석 리포트.
- **가치**: 돈을 더 내도 아깝지 않은 프리미엄 경험.

## 4. AI 활용 전략

**"AI는 차별화 수단이지, 수익화의 필수 조건은 아니다."**

- 초기에는 AI 없이 **"데이터 저장 및 관리(Local vs DB)"** 만으로도 충분히 수익화가 가능합니다.
- AI 기능(예문 생성, 분석 등)은 **Prompt Set(기획된 프롬프트 묶음)** 형태로 모듈화하여 필요한 곳에 API로 연결합니다.

## 5. 실행 체크리스트

- [ ] **정체성 확립**: "읽고 끝나는 영어가 아닙니다. 기억되는 영어입니다." (메인 카피 변경)
- [ ] **데이터 락인(Lock-in)**: Local Storage에 저장된 데이터를 DB로 옮기는 Migration UI/UX 구현.
- [ ] **가격 정책**: 기능별(저장 개수, AI 횟수 등) 무료/유료 제한 기준 수립.
