# 에이전트 스킬 가이드 (Agent Skills Guide)

이 프로젝트에는 AI 에이전트에게 전문적인 지식과 기능을 제공하는 **Vercel 에이전트 스킬**이 설정되어 있습니다. 이 스킬들은 코드 품질, 성능 최적화, 디자인 일관성을 보장하기 위한 강력한 도구입니다.

## 설치 및 설정 (Installation & Setup)

프로젝트에 새로운 스킬을 추가하거나 최신 버전으로 유지하려면 터미널에서 다음 명령어를 실행하십시오.

```bash
# Vercel 에이전트 스킬 추가 및 업데이트
npx add-skill vercel-labs/agent-skills
```

명령어를 실행하면 설치 가능한 스킬 목록이 나타납니다. 원하는 스킬을 선택하면 `.agent/skills/` 디렉토리에 관련 지침(`SKILL.md`)이 설치되며, 이후 에이전트가 대화 중에 해당 지침을 자동으로 참고하게 됩니다.

## 설치된 스킬

### 1. Vercel React 베스트 프랙티스 (Vercel React Best Practices)

**설명:**
Vercel 엔지니어링 팀에서 관리하는 React 및 Next.js 성능 최적화 가이드라인입니다. 8개 카테고리에 걸쳐 45개의 규칙을 포함하고 있습니다.

**주요 카테고리:**

1.  **Waterfall 제거 (Critical):** `async/await` 최적화, `Promise.all` 활용, Suspense를 이용한 스트리밍.
2.  **번들 크기 최적화 (Critical):** 임포트 관리, `next/dynamic` 활용, 조건부 로딩.
3.  **서버 측 성능 (High):** 캐싱 전략 (`React.cache`, LRU), 직렬화 최소화, 병렬 데이터 페칭.
4.  **클라이언트 데이터 페칭:** SWR을 통한 중복 요청 제거 및 이벤트 리스너 관리.
5.  **리렌더링 최적화:** 메모이제이션(Memoization), 안정적인 콜백, Transition 활용.
6.  **렌더링 성능:** SVG 최적화, `content-visibility` 활용, 하이드레이션 전략.
7.  **JavaScript 성능:** 효율적인 루프 처리, 캐싱, 데이터 구조 최적화.
8.  **고급 패턴:** Event handler refs 및 안정적인 callback refs.

**활용 명령어 예시:**

- "이 컴포넌트의 성능 문제를 리뷰하고 Vercel 가이드라인에 맞게 수정해줘."
- "`app/page.tsx`에서 비동기 Waterfall(병목)이 발생하는 지점을 찾아서 `Promise.all`로 최적화해줘."
- "이 파일에서 불필요한 리렌더링이 발생하는지 분석하고 `memo`나 `useMemo`가 필요한 곳을 알려줘."
- "클라이언트 컴포넌트의 번들 크기를 줄이기 위해 `next/dynamic`을 적용할 만한 곳이 있어?"

### 2. 웹 디자인 가이드라인 (Web Design Guidelines)

**설명:**
웹 인터페이스 가이드라인을 준수하는지 UI 코드를 검토합니다. 접근성(Accessibility), UX, 디자인 일관성을 체크합니다.

**작동 방식:**
Vercel의 공식 저장소에서 최신 가이드라인을 동적으로 가져와 검토를 수행하므로 항상 최신 기준을 유지합니다.

**활용 명령어 예시:**

- "내 UI 코드를 리뷰하고 웹 접근성(WAI-ARIA) 기준에 맞는지 확인해줘."
- "이 버튼 컴포넌트의 클릭 영역과 색상 대비가 모바일 사용자에게 적절한지 체크해줘."
- "전반적인 사용자 경험(UX) 관점에서 개선할 점이 있는지 디자인 감사를 해줘."
- "현재 페이지가 Vercel의 웹 인터페이스 베스트 프랙티스를 잘 따르고 있어?"

## 종합 활용 예시

에이전트에게 다음과 같이 구체적으로 요청하면 효과적입니다.

- **컴포넌트 생성 시:**
  > "새로운 `ExpressionCard` 컴포넌트를 만들어줘. Vercel의 React 성능 가이드라인을 준수하고, 스크린 리더 사용자를 위한 접근성 태그도 포함해줘."
- **성능 분석 시:**
  > "요즘 페이지 로딩이 느린 것 같아. `lib/expressions.ts`와 연결된 데이터 페칭 로직을 Vercel 베스트 프랙티스에 따라 검토해줘."
- **리팩토링 시:**
  > "이 복잡한 폼 컴포넌트를 리팩토링해줘. 특히 성능(리렌더링)과 디자인 가이드라인(일관된 여백 및 폰트)을 중점적으로 봐줘."

이 가이드라인은 대화 중에 자연스럽게 적용되지만, 특정 파일에 대해 명시적으로 검토를 요청하면 더욱 정교한 피드백을 받을 수 있습니다.
